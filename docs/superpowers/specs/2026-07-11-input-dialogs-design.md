# 应用内输入对话框设计

## 目标

移除饮水快捷量和训练模板命名流程中的浏览器原生 `prompt`，使输入、校验、取消和键盘操作符合应用自身的视觉与交互标准。

## 饮水快捷量

- 点击当前快捷量打开对话框，输入框预填现值并获得焦点。
- 允许范围为 50 到 2000 ml，步进为 50 ml。
- 不合法值留在对话框中并显示字段级错误；取消不修改设置。
- 保存后立即更新快捷按钮并持久化设置。

## 训练模板命名

- 仅在当前训练至少有一个有效动作时打开命名对话框。
- 默认名称来自训练标题，无标题时使用“常用训练”。
- 名称去除首尾空格，最长 40 字；空名称和已有模板重名会显示字段级错误。
- 保存成功后关闭对话框、持久化并刷新模板列表；取消不新增模板。

## 无障碍与验证

- 使用原生 `dialog`、显式标题、说明、标签和 `role=alert` 错误区。
- 表单支持 Enter 提交，取消按钮是安全默认操作。
- 浏览器测试覆盖打开、默认焦点、取消、非法值、成功保存、重名拦截和手机无溢出。
