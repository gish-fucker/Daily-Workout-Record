# Pro 纵向进展报告实施计划

## 目标

在保留免费 28 天报告的基础上，为服务器验证的 Pro 用户增加本机生成的 90 天与年度纵向报告，并为未登录、Free、权益失败和数据不足提供真实状态。

## 实施顺序

1. 在洞察页新增独立“长期进展”区域，静态或未配置权益部署默认隐藏。
2. 新增 90 天/年度运行时周期选择，不写入业务状态或本地存储。
3. 实现纯派生长期窗口统计：训练、组数、状态覆盖、活跃周/月、最长连续训练周、睡眠与疼痛汇总。
4. 实现数据门槛和首尾阶段比较；数据不足时只给缺口，不生成趋势。
5. 接入现有账号权益状态：未登录引导至账号，Free 展示准备度和未开放说明，Pro 展示报告，失败状态不猜测层级。
6. 实现本地 Markdown 导出与剪贴板降级，排除备注、动作、重量、次数、具体训练日期和 ID。
7. 更新隐私、条款、README、版本号与 PWA 缓存。
8. 扩展浏览器冒烟：五种权益状态、90/365 门槛、跨年周/月、隐私导出、本地存储不变和 390px 布局。
9. 运行静态检查、完整冒烟、diff 检查和截图审查，再提交、推送并验证公网静态模式。

## 验收

- 免费 28 天报告不受影响。
- 只有服务器响应为 Pro 时，正常界面才展示长期报告内容。
- 任何长期报告计算和导出都不上传或修改本地记录。
- 静态站不展示权益、Pro 报告或购买入口。
- 数据不足、权益不可用和退出登录均不会泄露此前报告内容。
- 自动化检查、桌面与移动端视觉检查、公网在线与离线检查全部通过。
