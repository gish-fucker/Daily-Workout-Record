# 生产运行层设计

## 目标

让 Node 服务具备托管平台所需的配置校验、请求追踪、健康检查和优雅停止能力，同时保持零新增运行依赖。

## 配置

- `HOST` 默认 `127.0.0.1`，公开容器部署显式设置为 `0.0.0.0`。
- `PORT`、`ADVICE_RATE_LIMIT` 和 `UPSTREAM_TIMEOUT_MS` 在启动时校验范围，非法配置直接失败并给出明确错误。
- `TRUST_PROXY=1` 时才使用 `X-Forwarded-For` 的首个地址作为限流身份。
- `.env.example` 提供不含秘密的部署配置样例。

## 可观测性

- 每个响应附带服务生成的 `X-Request-Id`。
- API 请求完成后输出单行 JSON 日志，包含时间、请求 ID、方法、路径、状态码和耗时，不记录用户内容或网络地址。
- `/api/health` 返回 `status`、应用版本、运行秒数、AI 配置状态和模型名，不返回密钥。

## 生命周期

- 启动日志使用结构化 JSON，包含监听地址和版本。
- 收到 `SIGTERM` 或 `SIGINT` 后停止接受新连接，等待现有请求完成后退出。
- 超过 10 秒仍未关闭时强制退出并记录失败事件。

## 验证

- 自动化测试确认健康元数据、请求 ID、安全头和现有 API 行为。
- 完整浏览器回归继续覆盖本地服务、PWA 和业务流程。
