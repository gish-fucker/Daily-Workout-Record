# P1 第二天留存闭环设计

日期：2026-07-15  
状态：用户已确认，直接实施

## 目标

用户完成一次训练后，系统记住完成情况、生成可解释的下一次训练，并在下次打开时让用户直接开始。P1 同时采集未来个人规律分析所需的恢复与训练数据，但不输出统计结论。

## 范围与边界

- 包含：训练完成结果、下一次训练计划、首页优先入口、继续草稿、规律解锁进度、最小行为记录。
- 不包含：相关性图表、睡眠或疼痛预测、周期化训练、健康评分、AI 报告、付费与云同步。
- 现有训练记录仍只保存用户明确完成的组；跳过或未完成信息仅作为本次结果和后续建议的元数据，绝不伪造为训练完成。

## 数据模型

在既有本地状态上新增兼容字段 `nextWorkoutPlan`：

```js
{
  version: 1,
  id: "next_plan_...",
  sourceWorkoutId: "workout_...",
  createdAt: "ISO",
  scheduledFor: "YYYY-MM-DD",
  status: "planned", // planned | started | superseded
  title: "下一次训练",
  exercises: [{ name, metric, sets, adjustment, reason }],
  adjustments: ["..."],
  reasons: ["..."],
  source: "last_workout",
  acceptedAt: "ISO 或空",
  startedAt: "ISO 或空"
}
```

训练记录补充 `completionSummary`，保存本次明确完成、跳过和待完成组的计数；已有历史记录在读取时以零值兼容。恢复记录继续使用已有的睡眠、精力、酸痛、疼痛字段。计划上的 `acceptedAt`、`startedAt` 和来源训练日期提供 P2 所需的建议接受率、训练间隔和回访数据。

## 生成规则

生成计划时按优先级应用下列可见规则：

| 条件 | 下一次调整 | 向用户解释 |
| --- | --- | --- |
| 疼痛 >= 4 | 不安排原负荷训练，改为恢复训练 | 疼痛信号优先于进步 |
| 恢复差（睡眠 < 6 且酸痛 >= 4） | 减少每个动作一组，或改轻量方案 | 今天先把恢复放在前面 |
| 上次有未完成或跳过组 | 优先把该动作作为下次起点，保持或降低目标 | 先稳定完成上次没完成的部分 |
| 实际次数低于目标 | 保持重量，目标回到上次实际次数 | 先把动作完成得更稳 |
| RPE <= 4 且所有完成 | 重量小幅增加；无重量则增加 1–2 次 | 上次有余力，做一次小幅推进 |
| RPE >= 8 | 保持重量和组数 | 强度已经偏高，先保护恢复 |
| 其余情况 | 保持上次负荷和组数 | 上次节奏合适，先稳定复现 |

规则按来源训练的实际数据构造，因此每项建议都带有“调整”和“原因”。无可复用动作时才回退到现有的每日推荐模板。

## 页面与流程

1. 用户结束训练并填写主观感受后，点击唯一主动作“保存并安排下次训练”。
2. 系统保存训练、生成计划，再显示结果对话框：今天完成情况、完成/跳过/待完成数、下次训练日期、建议动作和理由。关闭结果页不会丢失计划。
3. 首页发现有效计划时，把原“今天练什么？”替换为“下一次训练”卡片，显示来源、动作数量、关键调整和主按钮“开始训练”。存在草稿时额外显示次级按钮“继续上次训练”。
4. 用户开始计划时，将计划标为已开始，使用计划动作直接进入现有逐组训练流程；完成后由新计划取代旧计划。
5. 首页和结果页展示“个人训练规律：X/7 天”。X 为同时至少有一项恢复状态和一次训练记录的不同日期数，上限 7；详情只列举已收集的数据类别，并说明会生成“初步分析”，不承诺发现规律。

## 兼容与错误处理

- 新字段在加载、导入和重置时都可缺省；缺省即没有下一次计划，不影响既有记录。
- 编辑或删除来源训练后，如其为当前计划来源则将计划标为过期并回退到每日推荐，避免展示不存在的依据。
- 计划动作缺失必要组数据时不启动，回退到每日推荐并说明原因。
- 低恢复与明显疼痛仍遵循既有安全提示，建议不是医疗诊断。

## 验收标准

- 首次完成训练后可看到结果页和一份带理由的下一次训练计划。
- 次日打开首页，计划优先出现，用户无需动作库或重新配置即可开始。
- 低恢复、疼痛、未完成、轻松完成、吃力完成均产生符合规则的可解释建议。
- 训练记录、恢复记录、计划接受和开始时间在本地状态中可用；规律入口只显示进度和数据采集状态。
- 桌面和 390px 宽度下没有横向溢出，现有四项导航及 P0 逐组完成规则持续可用。

## 自检

本设计将即时留存闭环作为 P1 唯一完整功能；规律洞察只保留数据和预期。没有未定义占位项、没有与“只保存明确完成组”冲突的写入方式，也没有把任何长期结论提前承诺给用户。
